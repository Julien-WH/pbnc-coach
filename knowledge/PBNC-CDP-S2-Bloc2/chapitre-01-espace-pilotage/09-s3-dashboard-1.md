# S3 - Construire un dashboard de pilotage (1)

**Type :** E-learning
**Durée :** ~30 min
**Statut :** ✅ Complété

## Points clés à retenir

1. **Qu'est-ce qu'un dashboard de pilotage ?** Un tableau de bord qui centralise les indicateurs clés du projet pour permettre au chef de projet (et aux parties prenantes) de voir l'état du projet en un coup d'œil.

2. **Les principes de design d'un bon dashboard** :
   - **Hiérarchie visuelle** : les informations critiques sont les plus visibles
   - **Simplicité** : moins de widgets, plus d'impact
   - **Contexte** : chaque chiffre a besoin d'un point de comparaison (objectif, période précédente)

3. **Les composantes d'un dashboard de pilotage projet** :
   - Avancement global (% de complétion)
   - Budget consommé vs prévu
   - Vélocité de l'équipe (stories points livrés par sprint)
   - Indicateurs qualité (taux de bugs, taux de satisfaction)
   - Risques actifs

4. **Pour qui est le dashboard ?** Il faut distinguer :
   - Dashboard **opérationnel** (équipe) : détail des tâches, blocages
   - Dashboard **stratégique** (direction/sponsor) : synthèse, ROI, risques majeurs

5. **Outils no-code recommandés** : Notion, Airtable (avec vues), Google Looker Studio, Databox.
