# S3 - Construire un dashboard de pilotage (4)

**Type :** E-learning
**Durée :** ~35 min
**Statut :** ✅ Complété

## Points clés à retenir

1. **Industrialiser le rituel de pilotage** : Le dashboard n'a de valeur que s'il est consulté régulièrement dans un cadre défini. Créer un rituel hebdomadaire de revue des indicateurs.

2. **La revue de sprint comme moment de pilotage** : Intégrer la lecture du dashboard dans la rétrospective de sprint pour connecter les données aux décisions d'ajustement.

3. **Évoluer le dashboard avec le projet** : Les indicateurs pertinents en phase de discovery ne sont pas les mêmes qu'en phase de déploiement. Prévoir une revue trimestrielle des KPIs affichés.

4. **Cas fil rouge - Mise en pratique** : Application des concepts vus dans les 4 sessions sur un projet concret : définir les KPIs, configurer les sources de données, construire les vues, définir les alertes.

5. **Le dashboard comme outil de confiance** : Un dashboard bien construit et régulièrement tenu renforce la confiance des parties prenantes. Il montre que le projet est piloté avec méthode, pas géré au feeling.

6. **Synthèse du Chapitre 1** :
   - Roadmap agile = savoir où on va avec flexibilité
   - Budget dynamique = voir les coûts en tendances, pas en instantané
   - Dashboard de pilotage = tableau de bord actionnable des indicateurs clés
