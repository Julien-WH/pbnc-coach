# S1 - Le cockpit : voir juste pour décider tôt

**Type :** E-learning
**Durée :** ~35 min
**Statut :** ✅ Complété

## Points clés à retenir

1. **"Voir juste" avant "voir tout"** : Trop d'information tue l'information. Le cockpit efficace affiche ce qui est actionnable, pas tout ce qui est mesurable.

2. **Décider tôt réduit le coût des erreurs** : Plus on détecte un écart tôt dans le projet, plus il est facile et peu coûteux de corriger. Attendre la fin du sprint pour regarder les indicateurs revient à piloter en regardant dans le rétroviseur.

3. **Les trois types d'indicateurs à surveiller** :
   - **Avancement** : le projet progresse-t-il au rythme prévu ?
   - **Qualité** : ce qu'on livre correspond-il aux attentes ?
   - **Budget** : les ressources sont-elles consommées de manière cohérente ?

4. **Le rôle de l'IA dans le cockpit** : Les outils IA peuvent automatiser la collecte et la synthèse de données (comptes rendus, tickets, commits) pour alimenter le cockpit sans sur-solliciter l'équipe.

5. **Éviter la sur-instrumentation** : Un cockpit avec 50 indicateurs est aussi inutile qu'un cockpit avec 0. La règle : chaque indicateur doit déclencher une action possible.
