# S3 - Construire un dashboard de pilotage (2)

**Type :** E-learning
**Durée :** ~30 min
**Statut :** ✅ Complété

## Points clés à retenir

1. **Choisir ses sources de données** : Un dashboard ne vaut que par la qualité et la fraîcheur de ses données. Identifier les sources (Jira, Notion, Airtable, Google Sheets) et les fréquences de mise à jour est la première étape.

2. **L'automatisation des flux de données** : Utiliser Make.com ou Zapier pour alimenter automatiquement le dashboard à partir des outils de l'équipe. Évite la saisie manuelle qui est source d'erreurs et de délais.

3. **Construire les vues adaptées** :
   - Vue **temps réel** : pour les indicateurs opérationnels
   - Vue **hebdomadaire** : pour le suivi de sprint
   - Vue **mensuelle** : pour le reporting direction

4. **Intégrer les alertes** : Le dashboard doit non seulement afficher mais alerter. Configurer des notifications automatiques quand un indicateur dépasse un seuil critique.

5. **Tester le dashboard avec l'équipe** : Un bon dashboard est un dashboard utilisé. Le tester avec les futurs utilisateurs permet de valider que les indicateurs sont compris et actionnables.
