# S3 - Construire un dashboard de pilotage (3)

**Type :** E-learning
**Durée :** ~30 min
**Statut :** ✅ Complété

## Points clés à retenir

1. **Mise en situation pratique** : Construction pas à pas d'un dashboard de pilotage pour un projet no-code type, avec Airtable ou Notion comme base de données.

2. **Les 5 blocs d'un dashboard projet complet** :
   - **Bloc 1 - Avancement** : roadmap, % complétion par jalon
   - **Bloc 2 - Budget** : consommé, disponible, projeté
   - **Bloc 3 - Équipe** : vélocité, charge, disponibilité
   - **Bloc 4 - Qualité** : satisfaction utilisateur, dette technique, incidents
   - **Bloc 5 - Risques** : risques actifs, plans d'atténuation en cours

3. **Structurer les données correctement en amont** : Un dashboard ne peut pas être meilleur que les données qui l'alimentent. Définir les conventions de saisie et les responsabilités de mise à jour.

4. **Intégration IA** : Utiliser des outils comme Notion AI ou des scripts Make pour synthétiser automatiquement les statuts d'équipe en points d'avancement lisibles pour la direction.

5. **Partage et accès** : Définir qui voit quoi. L'équipe voit les données opérationnelles, la direction voit la synthèse, les clients voient uniquement ce qui est pertinent pour eux.
