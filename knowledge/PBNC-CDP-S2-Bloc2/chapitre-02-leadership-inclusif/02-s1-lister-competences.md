# S1 - Lister les compétences nécessaires

**Type :** E-learning
**Durée :** ~35 min
**Statut :** ✅ Complété

## Points clés à retenir

1. **Commencer par les besoins du projet, pas par les personnes disponibles** : La liste des compétences nécessaires doit être définie à partir du périmètre fonctionnel et technique du projet, indépendamment de qui est disponible.

2. **Types de compétences à identifier** :
   - **Compétences techniques** : développement no-code, data, design UX, API...
   - **Compétences métier** : connaissance du secteur, processus client...
   - **Compétences transversales** : gestion de projet, communication, facilitation...

3. **Granularité adaptée** : Être suffisamment précis pour être actionnable ("maîtrise de Make.com pour les automatisations de niveau intermédiaire") sans être trop détaillé.

4. **Distinguer compétences critiques et souhaitables** : Toutes les compétences ne sont pas au même niveau d'importance. Identifier les compétences bloquantes vs les compétences qui seraient un plus.

5. **Anticiper l'évolution des besoins** : Les compétences nécessaires en phase de build ne sont pas les mêmes qu'en phase de run. Prévoir les besoins futurs dans la cartographie.
