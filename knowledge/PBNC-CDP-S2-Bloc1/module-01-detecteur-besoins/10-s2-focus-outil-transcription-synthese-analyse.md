## Slide 1 : Présentation des différents outils

### Outils de transcription/analyse

| Outil | Points clés |
|-------|-------------|
| **Leexi** | Identification automatique des intervenants, résumés personnalisables, détection prochaines actions, intégration outils gestion de tâches |
| **Otter.ai** | Transcription temps réel, recherche puissante, export multi-format |
| **Fireflies** | Automatisation complète, intégration native Zoom/Google Meet |
| **Rêve.AI** | Excellence transcription anglais + jargon technique/médiatique |
| **Notta** | Outil minimaliste, transcription, résumé IA, mots-clés (PAS liste de tâches) |
| **Meetgeek** | Résumés automatiques, suivi actions, intégration Teams/Zoom |

### Quiz slide 1 (tous corrects)

1. Outil le plus adapté pour jargon spécialisé → **Rêve.AI** ✓ (carte mystère)
2. Outils avec intégrations faciles → **Leexi, Meetgeek, Fireflies** ✓ (PAS Otter)
3. Fonctionnalité commune Leexi+Meetgeek → **suivi des prochaines actions** VRAI ✓ (autres FAUX)
   - FAUX : "reconnaissance des intervenants" (Leexi uniquement), "intégration gestion de tâches" (Meetgeek = Teams/Zoom seulement)

## Slide 2 : Utiliser Notta pendant une réunion en visioconférence

### Fonctionnalités Notta

- Transcription en temps réel
- Résumé IA automatique
- Extraction des mots-clés
- **PAS** de liste de tâches automatique

### Compatibilité Notta

- ✅ Zoom
- ✅ Google Meet
- ✅ Microsoft Teams
- ❌ Skype (incompatible)

### Quiz slide 2 (tous corrects)

1. Après une réunion Notta → **Résumé IA + mots-clés + transcription complète** ✓ (PAS liste de tâches)
2. Plateformes compatibles → **Zoom, Google Meet, Microsoft Teams** ✓ (PAS Skype)
3. Cas d'usage UX designer/chercheur → **Analyse verbatim entretiens utilisateurs** VRAI ✓
   - FAUX : Documentation des décisions en réunion d'équipe (cas d'usage générique, pas UX-spécifique)

### Renforcement slide 2

Plateformes Notta : Skype=FAUX, Microsoft Teams=VRAI, Google Meet=VRAI, Zoom=VRAI

## Slide 3 : Utiliser Otter.ai pendant une réunion en présentiel

### Points clés Otter.ai

- Spécialement adapté aux **réunions physiques** (contrairement aux outils vidéoconférence)
- Usage : placer smartphone/ordinateur au centre de la table
- Génère : transcription complète + résumé points clés + mots-clés
- Permet de rester concentré sur la discussion

### Bonnes pratiques Otter.ai

1. **Prévenir** les participants de l'enregistrement
2. Utiliser un **bon microphone** pour meilleure qualité sonore
3. **Nommer et organiser** les enregistrements pour les retrouver facilement

### Positionnement Otter.ai

Similaire à Notta, Fireflies, etc. Différences : précision retranscription, options personnalisation, richesse intégrations.

### Quiz slide 3 (tous corrects)

1. Options post-réunion Otter.ai → **Toutes les 4** ✓ :
   - Exporter différents formats (PDF, DOCX...)
   - Générer résumé automatique avec points clés
   - Ajouter commentaires/surlignages a posteriori
   - Partager document avec collègues
2. Bonne pratique en présentiel → **Renommer et organiser les enregistrements** ✓
   - FAUX : "Ne pas se soucier d'identifier les intervenants" (Otter n'identifie pas automatiquement en présentiel)
